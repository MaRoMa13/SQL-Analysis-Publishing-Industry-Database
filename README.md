# SQL-Analysis-Publishing-Industry-Database
This notebook presents a SQL-based analysis of a relational publishing database to answer key business questions related to books, authors, publishers, and user engagement, supporting data-driven product and strategy decisions.

## Industry
Publishing / Digital Products

## Business Problem
A startup in the digital publishing market required insights from historical data to define a value proposition for a new product focused on book readers.

## Objective
Analyze books, authors, publishers, and user reviews to extract business-relevant insights using SQL.

## Data Description
Relational database including:
- Books
- Authors
- Publishers
- User ratings and text reviews

## Tools Used
- SQL (PostgreSQL-style queries)
- Relational Databases

## Methodology
- Explored database schema and relationships between tables.
- Wrote complex SQL queries using JOINs, subqueries, aggregations, and filters.
- Calculated metrics such as average ratings, number of reviews, and publication volume.
- Applied business rules to filter relevant data (e.g., minimum page count, minimum number of ratings).

## Key Findings
- Identified publishers with the highest volume of substantial publications.
- Determined authors with the highest average ratings based on sufficient user feedback.
- Analyzed user engagement through review behavior.

## Business Impact
The analysis provides insights to:
- Support product strategy decisions
- Identify high-performing authors and publishers
- Understand user engagement patterns

## Challenges & Learnings
- Optimized SQL queries for accuracy and readability.
- Strengthened analytical thinking by translating business questions into efficient SQL logic.
- Reinforced attention to data quality and filtering criteria.
